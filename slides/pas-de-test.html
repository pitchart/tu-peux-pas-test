Les tests étaient bloquants, nous les avons abandonnés

  • Des temps d'exécution trop longs : trop de TF, pas assez de TU
  • Fort couplage dans le code, une simple modification demande un refactoring de plusieurs tests

Même avec une forte couverture de code, nous avions des bugs

  • Ne jamais TOUT tester
  • Rester pragmatique dans l'approche
  • 100% de couverture n'évite pas les bugs

Il n'y avait pas de test sur le projet de toute façon

  • Incorporer des tests avant les évolutions de code
  • Incorporer des tests fonctionnels sur les actions clés
  • Incroporer des tests sur les nouvelles fonctionnalités
